## 💡 使用建议

### 推荐工作流
1. **早上到公司/家**：先 `git pull` 拉取最新代码
2. **开始开发**：`npm run dev` 启动项目
3. **完成工作**：`git push` 推送代码
4. **换电脑**：在新电脑上 `git pull` 继续工作

### 避免冲突
- ✅ 每次开始工作前先 `git pull`
- ✅ 每次结束工作后立即 `git push`
- ❌ 避免同时在两台电脑上修改同一个文件
- ❌ 不要长时间不推送代码

## 🆘 遇到问题？

### 常见问题速查
1. **npm install 很慢**
   ```bash
   npm config set registry https://registry.npmmirror.com
   npm install
   ```

2. **Git 推送失败**
   - 检查网络连接
   - 确认 GitHub 仓库地址正确
   - 确认已登录 GitHub 账号

3. **代码冲突**
   - 先 `git pull` 拉取最新代码
   - 手动解决冲突（编辑器会标记）
   - `git add .` → `git commit` → `git push`

4. **API 配额用完**
   - 访问 https://ai.google.dev/ 升级付费计划
   - 或申请新的免费 API Key
